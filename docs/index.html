First launch

Opening the app the first time

SSH Files Viewer is open-source and ad-hoc signed — there's no paid Apple Developer ID — so macOS Gatekeeper blocks it the first time you open it. After moving the app into your Applications folder, do one of these (you only need to do it once):

Option A — Terminal (quickest)

/usr/bin/xattr -dr com.apple.quarantine /Applications/SSHFilesViewer.app

Then open the app normally from Launchpad or Applications.

Option B — System Settings

  1. Double-click the app. When macOS says it can't verify the developer, click Done.
  2. Open System Settings → Privacy & Security.
  3. Scroll to the Security section and click Open Anyway next to the “SSHFilesViewer was blocked” message.
  4. Click Open to confirm. The app launches normally from then on.

Features

🗂️

Full remote browser

Breadcrumb navigation, back/forward/up/home, live filtering, and color-coded file-type icons with real language logos.

👁️

Rich previews

Syntax-highlighted code, inline images, and CSV/TSV rendered as a real spreadsheet — in a side panel or its own window.

✏️

Edit & save

Edit text files in place with explicit saving (⌘S), an unsaved-changes indicator, and a save-on-close prompt. No autosave.

🎨

8 color schemes

One Dark, Dracula, Nord, Monokai, Gruvbox, Solarized (dark & light), and GitHub Light — with adjustable font size and wrapping.

⬇️

Download anything

One-click download with live progress, or open files in their default macOS app.

🔒

Secure by design

Reuses your existing SSH config, keys, and agent. Passwords stay in the macOS Keychain — never on a command line.

Why it's robust

Zero third-party runtime dependencies — it drives the system ssh binary, so it transparently works with your ~/.ssh/config, keys, agent, known_hosts, and jump hosts. Connection multiplexing keeps listing, preview, and download fast. A careful, portable listing parser preserves exact filenames across Linux and BSD remotes.

Build from source

# Requires macOS 13+, Xcode command-line tools, and ssh (ships with macOS)
git clone https://github.com/Alyetama/sshFilesViewer.git
cd sshFilesViewer
./run.sh # builds a release .app and launches it
